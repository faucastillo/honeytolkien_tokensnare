HoneyTolkien - Servidor de Alertas

Este servidor web simple se utiliza para recibir los "call home" de los honeytokens generados por la herramienta tokensnare-cli desarrollada para el TP de la materia de Seguridad de Información por el equipo HoneyTolkien.

{{ active_tokens }} Tokens Activos
{{ hits }} Hits Totales Recibidos

Rutas de la Interfaz Web:

  • GET / — Página de inicio (esta página) que muestra el total de tokens y hits.
  • GET /tokens — Lista de los tokens registrados con un pequeño resumen de info de cada uno.
  • GET /tokens/<token_id> — Detalles de un token especifico.
  • GET /website_demo — Sitio web de demostración (HoneyBank) para testing de honeytokens de clonación.

Endpoints de la API (Administración):

  • POST /api/tokens — Registrar un nuevo honeytoken. (JSON Req.)
  • GET /api/tokens — Listar todos los honeytokens registrados. (JSON Resp.)
  • GET /api/tokens/<token> — Mostrar detalles, logs, e historial de hits de un token específico.
  • DELETE /api/tokens/<token> — Eliminar un honeytoken específico y sus hits asociados.
  • DELETE /api/tokens/all — Eliminar todos los tokens y hits (reseteo de la base de datos).
  • OPTIONS /api/callback — CORS preflight para reportes de clonación por JS.
  • POST /api/callback — Recibir reporte de sitio web clonado (JS).

Endpoints de Tracking (Honeytokens):

  • GET /image/<token>.png — Imagen 1x1 PNG transparente usada como beacon para rastreo.
  • GET /link/<token> — Endpoint de rastreo tipo enlace. Registra hit y retorna 204 No Content.
